# Console — Project State

*Read this file first in a new chat instead of scanning the whole project — it's the compact version. Update it whenever a phase ships, a decision changes, or an open item gets resolved. Everything here should stay short enough to be cheap to load; details live in the linked docs.*

## What it is

Console: a personal, fully offline productivity app — Tasks, Schedule, Habits, Finance, Focus Sessions, plus Analytics/Insights. Vanilla HTML/CSS/JS, no framework, no build step, IndexedDB, must open directly from `file://`. Full feature/data-model spec: `docs/Console_Features_List.md`.

## Locked decisions (don't relitigate without a real reason)

- No streaks (Habits) · strict 8 event types (Schedule) · YNAB envelopes (Finance) · GTD inbox-first (Tasks) · fully offline · 3 fonts only (Source Serif 4 / Inter / JetBrains Mono) · two-voice stat rule.
- Classic `<script>` tags, not ES modules — `type="module"` import graphs get blocked by CORS under `file://` in several browsers.
- Build workflow: extraction-not-recreation (reuse prototype markup/CSS, don't redraw from description), one phase built and fully verified before the next starts. Full reasoning + drift log: `docs/Console_Workflow.md`.
